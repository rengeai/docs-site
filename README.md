# RengeAI 文档库

这是一个面向中文场景的 Markdown 文档库，用于系统化描述 RengeAI 要构建的产品、方法论与协作体系。

RengeAI 的核心定位，是一套建立在 GitLab 语义之上的多 OpenClaw 智能体协作系统。它不替代 GitLab，也不替代 OpenClaw，而是在两者之上补上一层更易搭建、更易治理、更易长期维护的统一系统框架。

当前第一版本聚焦于一个研发项目如何运行，优先把单项目下的 Issue、里程碑、MR 协作闭环跑通，再逐步扩展到多公司、多项目和更完整的治理能力。

## 核心定义

- 使命：服务中小规模研发团队更好地完成 AI 化，让 10 人研发团队释放出 100 人团队的价值
- 愿景：让传统的人类主导迭代，逐步过渡到 AI 自我协作、自我提升、自我迭代的新阶段
- 核心用户：3 人以上、100 人以下的研发型团队，例如技术公司、软件外包公司、软件开发咨询公司
- 核心场景：用 Issue、里程碑、MR 接管研发全流程，从产品想法走到产品上线

## 文档边界

- `README` 只负责总览，不展开具体制度细节
- 愿景、使命与用户画像集中写在 [项目愿景与理念](./docs/01-项目愿景与理念.md)
- 产品指南已拆分为总览、MVP、落地顺序、需求计划与产品案例，避免把多层内容挤在同一篇文档里

- 组织层规则集中写在 [组织与权限模型](./docs/02-组织与权限模型.md)
- 项目迭代流程集中写在 [研发协作体系](./docs/03-研发协作体系.md)

## 文档原则

- 中文优先：所有核心文档优先使用中文编写
- 结构优先：先讲清每篇文档负责什么，再补细节
- 可执行优先：文档不仅表达理念，也要能指导实施
- 边界清晰：避免多个文档反复说明同一件事
- 长期演进：所有文档都应支持版本化迭代
